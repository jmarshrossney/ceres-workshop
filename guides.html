Python tools¶

Getting help

It’s well worth knowing from the get-go how to bring up documentation for Python objects. Generally, you can do this by typing help(object), but in IPython you can also use ?.

$ python

>>> import math

>>> help(math.ceil)
Help on built-in function ceil in module math:

ceil(x, /)
    Return the ceiling of x as an Integral.

    This is the smallest integer >= x.

or in IPython

$ ipython

In [1]: import math

In [2]: math.ceil?
Signature: math.ceil(x, /)
Docstring:
Return the ceiling of x as an Integral.

This is the smallest integer >= x.
Type:      builtin_function_or_method

Testing

Install PyTest into your Conda environment

conda install pytest

Let’s say you have organised your code into a package package as follows:

├── package
│   ├── __init__.py
│   ├── module_1.py
│   ├── module_2.py
│   └── tests
│       ├── __init__.py
│       ├── test_module_1.py
│       └── test_module_2.py
└── pyproject.toml

where test_module_1.py contains test functions whose names start with test_, e.g.

# inside test_module_1.py
from package.module_1 import function, variable

def test_function():
    result = function()
    assert result is True

def test_variable():
    assert variable == 42

We can run these tests from anywhere in the tree shown above by simply executing

pytest

Benchmarking

As well as being a Standard Library package, timeit is a magic method in IPython.

In Ipython or a Jupyter notebook it can be used to measure execution time as follows:

def my_func():
    """Some function I want to time."""
    ...

%timeit my_func()

Typically it will run the given source code many times to get an average time.

Formatting

Install Black into your Conda environment

conda install -c conda-forge black

or

python -m pip install black

If you want to format Jupyter notebooks,

python -m pip install black[jupyter]

To format your code, just run the following command in the directory containing the files you want to format:

black .

You can configure black (e.g. change the max line length, tell it to ignore directories…) in your pyproject.toml file. See the documentation for details.

Linting

Install Flake8 into your Conda environment

conda install flake8

Run it as follows:

flake8 .

You may want Flake8 to ignore certain errors that you don’t care about. In this case, you need to create a file .flake8 (in your repository root) to configure Flake8. See the documentation.

Basic Git workflow¶

  1. Make a local clone of a remote repository

git clone https://github.com/<username>/<repo_name>.git

  1. Switch to an existing branch

git switch <existing_branch>

  1. Create a new branch, based on the current one, and switch to it

git checkout -b <new_branch>

  1. Inspect the local changes you’ve made

git diff --compact-summary
git status

  1. Add changes to the staging area

git add <files>

  1. Create a local commit based on staged changes

git commit -m "a message about what this commit contains"

  1. Push the changes to the remote

git push -u origin <new_branch>

  1. Merge your branch with the original one (or, alternatively, use a pull request!)

git switch <existing_branch>
git merge <new_branch>
